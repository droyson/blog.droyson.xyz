CSS Positions

In Creating a website, Part 2: Design I skimmed over the CSS part of the code providing a bare minimum of what I implemented. Based on popular feedback (50% of total 2 feedbacks) I decided to take a dive into the basics of CSS positioning. In this post we will take a look at different types of positioning and how to implement those with working examples.

There are 5 different position values - static, relative, absolute, fixed and sticky and 4 helper properties to position the elements - top, bottom, left and right. Let us explore each of these in detail.

Position - static

This is the default position given to every HTML element. One could also call this the unpositioned position (CSS irony!) because the helper properties have no effect on the placement of the element and it will always be placed in the given order. Below is an example from CodePen illustrating the same.

See the Pen Position Demo - Static by Royson (@droyson) on CodePen.

As can be seen above the Green box has not shifted even after providing top and left values.

Position - relative

Relative position as the name suggests will place the element relative to its current position. Changing the placement of an element will move the element but will retain the original position meaning the siblings to a relatively positioned element will act as if the element is still occupying its original space on the page. The example below will help better understand this concept.

See the Pen Position Demo - Relative by Royson (@droyson) on CodePen.

Green box has moved from its original position by 50px from top and left of it's current position but the Red box remains at it's original space.

Position - absolute

Absolute positions are similar to relative in the sense elements are positioned relative to the positioned parent. What is a positioned parent? Any immediate ancestor element to the absolute element that is not statically positioned is a positioned parent. If there are no positioned parent then the element is placed relative to <body> tag.

An absolute positioned element also gives up the space that it was supposed to occupy originally meaning the siblings to absolute element will act as if the element is not present and move to claim the blank space. Below is a demo to explain this.

See the Pen Position Demo - Absolute by Royson (@droyson) on CodePen.

The Green box is shifted by 50px from it's parents top and left and Red box has moved up to the space of Green box

Position - fixed

Fixed element is fixed to the viewport(screen). Nothing more to elaborate here. Check the example below

See the Pen Position Demo - Fixed by Royson (@droyson) on CodePen.

In the example above the Green box is not fixed to the screen cause the example is rendered with an <iframe> which has it's own viewport. Hence you would be able to see the Green box being fixed to iframe.

Position - sticky

As the name suggests the element sticks to the given position. The element behaves as if it's relatively placed until it scrolls to the specified position on the page post which it acts as a fixed element. In the example below Red box scrolls to top 20px and then stays there while the lower Green box scrolls under it.

See the Pen Position Demo - Sticky by Royson (@droyson) on CodePen.


That's all the basics of positioning in CSS.

Creating a website, Part 2: Design

In the previous post we discussed about hosting a static page on github with a custom domain. Now let's get started with one of the most important part of a website - Design. I call it the important part of a website cause - Jo dikhta hai, wahi bikta hai ( Loosely translated by google as - What you see sells ).

To have some clarity on the page structure I decided to move About me to a separate page.

The next step for me was creating a logo and favicon for the site. There are multiple file formats to create a logo but SVG is my go to choice for the simple reason that it is a vector graphic is not prone to pixelation. Favicon in the required format can be generated from SVG file using one of the many online converters.

It was a tedious task as I spent an entire day on designing the logo. I used inkscape to create the SVGs. It's free and easy to use once you get over the learning curve ( Which was 4+ hours for me 😛 ). The official tutorials provided by inkscape are easy to understand and get started with.

The first rule of webpage design is - Mobile first. Why? Cause,

  1. mobile usage is higher compared to desktop and
  2. scaling up from mobile layout to desktop is easier than scaling down.

To achieve this switch to mobile device with Device Mode.

Note: To move on with the rest of the section make sure you understand some basic CSS.

Before coding it is better to have a visual in hand. I did a quick pen and paper sketch before implementing it. Below is a rough sketch of what I had in mind.

As per this sketch, I needed the header to be fixed on top. The intuitive approach is to use position: fixed but then you would be trying to fix the margin since the content is now overlapping. The simpler way to go about is to use position: sticky and bam! it works like magic.

Each post needed to have a card like feel to it. To do that add a max-width on the container for each posts and some box-shadow to visually raise the content from the rest of the page. Center align the cards for larger screens using margin: auto style.

Few pointers on creating a responsive page.

  • Use Normalize.css to render consistent looking elements.
  • Set box-sizing: border-box on all elements using wildcard (*)
  • Set the document level font-size in pixels (px) and then use rem for setting the font-size of the elements. The following blog - Responsive Typography: rem, em, and px - describes this use case beautifully.
  • Always have a max-width for the main content. This ensures that while viewing on screens with higher resolution a user doesn't have to move his head across the length of the screen.

Creating a website, Part 1: Hosting

I'll be using this blog as an example for this post. The idea here is to evolve the blog with every new post in this series. So please don't judge me on how the blog appears, it might very will be a work in progress.

For this blog I wanted to host it on my personal domain so step 1 would be purchasing a domain name. This can be skipped if you do not want a custom domain name. I got mine from GoDaddy.com but you may use one of the many DNS providers available. Purchasing a domain is as simple as creating an account and making the payment and voila you have your domain name ready to be used.

Second step is to create a repository on github. In here create a file named index.html and type in the html. Having trouble writing html? Copy the below example.

I'm assuming basic knowledge of git commands here.

Once you have committed and pushed the changes to your remote github repository go to "Settings" tab in github and scroll down to Github Pages section. Update the branch and directory dropdowns as shown in the screenshot below and hit save.

A few seconds of github magic and boom! you have a live webpage. The link to your page will be available in the "Github Pages" section.

Next step would be pointing the domain to this new page. Now that you have the page live you should be able to see the "Custom domain" section. Add the domain or subdomain that you need to point to your webpage and click save. Now go to Manage DNS settings page provided by your DNS provider (GoDaddy.com in my case).
