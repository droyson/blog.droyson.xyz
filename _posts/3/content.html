In Creating a website, Part 2: Design I skimmed over the CSS part of the code providing a bare minimum of what I implemented. Based on popular feedback (50% of total 2 feedbacks) I decided to take a dive into the basics of CSS positioning. In this post we will take a look at different types of positioning and how to implement those with working examples.

There are 5 different position values - static, relative, absolute, fixed and sticky and 4 helper properties to position the elements - top, bottom, left and right. Let us explore each of these in detail.

Position - static

This is the default position given to every HTML element. One could also call this the unpositioned position (CSS irony!) because the helper properties have no effect on the placement of the element and it will always be placed in the given order. Below is an example from CodePen illustrating the same.

See the Pen Position Demo - Static by Royson (@droyson) on CodePen.

As can be seen above the Green box has not shifted even after providing top and left values.

Position - relative

Relative position as the name suggests will place the element relative to its current position. Changing the placement of an element will move the element but will retain the original position meaning the siblings to a relatively positioned element will act as if the element is still occupying its original space on the page. The example below will help better understand this concept.

See the Pen Position Demo - Relative by Royson (@droyson) on CodePen.

Green box has moved from its original position by 50px from top and left of it's current position but the Red box remains at it's original space.

Position - absolute

Absolute positions are similar to relative in the sense elements are positioned relative to the positioned parent. What is a positioned parent? Any immediate ancestor element to the absolute element that is not statically positioned is a positioned parent. If there are no positioned parent then the element is placed relative to <body> tag.

An absolute positioned element also gives up the space that it was supposed to occupy originally meaning the siblings to absolute element will act as if the element is not present and move to claim the blank space. Below is a demo to explain this.

See the Pen Position Demo - Absolute by Royson (@droyson) on CodePen.

The Green box is shifted by 50px from it's parents top and left and Red box has moved up to the space of Green box

Position - fixed

Fixed element is fixed to the viewport(screen). Nothing more to elaborate here. Check the example below

See the Pen Position Demo - Fixed by Royson (@droyson) on CodePen.

In the example above the Green box is not fixed to the screen cause the example is rendered with an <iframe> which has it's own viewport. Hence you would be able to see the Green box being fixed to iframe.

Position - sticky

As the name suggests the element sticks to the given position. The element behaves as if it's relatively placed until it scrolls to the specified position on the page post which it acts as a fixed element. In the example below Red box scrolls to top 20px and then stays there while the lower Green box scrolls under it.

See the Pen Position Demo - Sticky by Royson (@droyson) on CodePen.
